# 协作与 Issue 说明

本仓库默认使用中文沟通。

- 新开 issue、后续跟进、代码审查回复，默认都用中文。
- 如果你更方便用英文描述问题，也可以直接写英文；仓库这边会尽量用中文回复，必要时补少量英文关键词帮助定位。

## 报 Bug 时请尽量带上这些信息

- 进入路径：启动器、字海二级菜单、战斗内、结算页等。
- 设备与环境：手机 / 平板 / 桌面，系统版本，浏览器或 WebView 环境（例如微信内打开、Safari、Chrome）。
- 复现步骤：尽量写成 1、2、3 的顺序。
- 实际结果：你看到的报错、白屏、卡死、按钮无响应等。
- 预期结果：你原本希望发生什么。
- 附件：截图、录屏、控制台报错文字。

## 功能建议

- 先说明你想解决的具体问题，再描述希望的体验或效果。
- 如果只是方向讨论，也可以直接开 issue，不需要先准备完整方案。

## 提交改动

- 优先做小而明确的改动，避免把无关文件一起改掉。
- 如果改动会影响移动端 / Web 导出，说明你做过的基本验证。
